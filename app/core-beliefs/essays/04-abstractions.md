# Abstractions Are Compression for Thought

"Chairness." There are countless ways to describe a chair — wooden, four legs, seat height, with a back, used for sitting. But at some point, you just have the concept. Singular. Reusable. Efficient. You don't reconstruct furniture primitives every time you enter a room.

Abstractions are compression for thought. They eliminate the need to re-derive every detail and let you operate at the right level of discourse. "Let's worry about the chair, not the concept of sitting."

The problem with human communication is that we don't share the same compression. You say "chair" and I understand the general idea, but the edges don't align perfectly. What about a stool? A beanbag? A tree stump? Each person's compression algorithm has different thresholds, different prototypes, different exclusions.

Building shared abstraction structures — explicit, inspectable, revisable — is how we talk about complex systems without drowning in definitions. This is what the Semantic Abstraction Graph is about: persistent handles for shared concepts, so we can build on common ground instead of re-establishing it every time.
